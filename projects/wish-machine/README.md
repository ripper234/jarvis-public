# Wish Machine

*The umbrella project behind Request for Startups.*

## What it is

Wish Machine is a project in itself.

It is Ron's system for:
- listing ideas he cares about
- building some himself
- inviting other people to build some
- funding or advising the strongest ones
- staying close as a design partner or power user when that is the best role

`Request for Startups` is the public-facing format.
`Wish Machine` is the broader operating model.

## Why it exists

A lot of good ideas do not need to become companies that Ron personally runs.

Wish Machine creates a place to:
- capture those ideas clearly
- improve them over time
- prototype selectively
- make the best opportunities legible to founders and collaborators
- route energy toward the ideas that show real pull

It is also a practical way to turn idea overload into a portfolio of explicit opportunities.

## A variant of a Venture Studio

Wish Machine is a **variant of a Venture Studio**, with a lighter and more AI-native structure.

A classic Venture Studio usually:
- starts companies internally
- provides a shared operating team
- recruits founders into that structure
- puts substantial studio ownership and process around each company early

Wish Machine is different in a few ways:
- it starts from **idea specs and prototypes**, not from a heavy central operating machine
- it is more **public and porous**: some opportunities are published openly
- not every idea has to become a company
- Ron's role can change per project: **builder, advisor, investor, power user, or matchmaker**
- some projects stay as ideas, some become collaborations, and some may become funded startups

So the closest framing is:

> **an AI-era, idea-first, lightweight Venture Studio**

## Why AI changes the equation

AI makes this model much more practical than older studio models.

Because of AI, Wish Machine can:
- prototype much faster
- reuse prompts, agent workflows, and tooling across multiple projects
- test several theses in parallel
- kill weak directions earlier
- push more energy toward ideas that show traction

When software becomes cheaper to build, the bottleneck shifts more toward:
- taste
- distribution
- founder fit
- domain insight
- execution discipline

That plays directly into this model.

## How it works

1. Collect ideas worth existing.
2. Write down the core thesis clearly.
3. Prototype selectively when that helps.
4. Publish the best opportunities in a legible way.
5. Attract founders, collaborators, or operators.
6. Decide Ron's role case by case.
7. Keep compounding the portfolio of ideas and learnings.

## Examples of Wish Machine style projects

Examples already in or around this orbit include:
- Balance
- Know Thyself
- Next-Level Spa
- Song-Aware Running Coach
- OpenClaw Playground

Some of these may stay internal longer.
Some may become public requests.
Some may become investable companies.

## Short version

Wish Machine is a lighter, more public, AI-native version of a Venture Studio.

It is a system for turning a large set of meaningful ideas into:
- clearer opportunities
- better prototypes
- better founder matching
- and, sometimes, real companies.
