How Ron set up Jarvis — a personal AI agent running 24/7 on OpenClaw.

What This Is

A persistent AI assistant that lives on a cloud server, communicates via Telegram, has a workspace with files and memory, runs background jobs, and manages GitHub repos autonomously.

Before You Start

Cost

LLM: Claude Opus ~$15/MTok input, $75/MTok output. Budget $5-20/day.

Hosting: Railway ~$5-10/month for always-on.

Optimize for quality, not cost. Your time is worth more than API tokens.
Security
  • Treat the server like a personal device — it has your API keys
  • Use fine-grained tokens with minimal permissions
  • Secrets in environment variables, never in workspace files
Model Choice

Claude Opus 4 for main interactions. Consider Sonnet for cron jobs. The model matters more than you think.

Persistence

Agents wake up fresh each session. Files are your memory. Design a memory architecture early and back up to GitHub from day one.

Setup Steps

1

Infrastructure (Railway)

Fork the OpenClaw template, deploy to Railway, add persistent volume at /data, set ANTHROPIC_API_KEY and OPENCLAW_GATEWAY_TOKEN.

2

Telegram Channel

Create a bot via @BotFather, add the token to config, set allowFrom to your user ID.

3

Identity Bootstrap

Send your first message. The assistant reads BOOTSTRAP.md, you choose a name and personality together. Delete the file after — the assistant is "born."

4

Workspace Structure

SOUL.md (personality), MEMORY.md (long-term), HOT.md (critical rules), memory/ (daily logs), scripts/ (guards).

5

Cron Jobs

Morning briefing, workspace backup (every 30 min), GitHub polling, nightly summary, weekly review. Start with 2-3, add as needed.

6

GitHub Integration

Create a bot GitHub account, fork repos, generate PATs for push + PR. Build a deterministic write guard in Python.

Safety Layers

Never give the bot direct write access. PRs only. Rules enforced by code, not prompts.
Write Guard

All GitHub mutations through deterministic Python. The LLM forgets rules; code doesn't.

Message Guard

Post-send hook validates outgoing messages. Auto-corrects missing PR links, stale references.

Attention Budget

10 free + 20 earnable credits daily. At zero, assistant reacts with 👍 only (capture mode). Prevents overwhelm.

Backup

Every 30 minutes to a private GitHub repo. Your workspace IS your assistant.

Lessons Learned

  1. ChatGPT said 10 minutes. It took 4 hours. Expect a full day.
  2. Text rules leak. Code doesn't. Safety at side-effect boundaries.
  3. Tiny PRs always. 1-2 lines, 10-second review.
  4. Design for recovery, not prevention. The assistant will break things.
  5. Attention is the scarcest resource. Not tokens. Not compute.
  6. Memory architecture > prompt engineering.
  7. Back up from day one.
