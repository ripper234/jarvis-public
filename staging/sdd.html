Spec-Driven Development

Write the spec. Let the AI write the code.

Spec-driven development (SDD) flips the traditional workflow: instead of writing code and documenting it later, you write specifications first and let AI agents generate the implementation. The spec becomes the source of truth, not the code.

This matters because AI coding agents are getting good enough to implement from specs, but they still need clear intent. Vibe coding produces throwaway prototypes. SDD produces maintainable systems where the human stays in control of what gets built and why, while the AI handles how.

Toolkits

GitHub Spec Kit Official

GitHub's open source SDD toolkit. CLI-based phases: constitution → specify → plan → tasks → implement. 40+ community extensions. Supports Claude, Copilot, Gemini, OpenAI.
OpenSpec YC-backed

Lightweight spec-driven framework. Three-phase state machine (propose/apply/archive) with spec deltas. Compatible with 20+ AI coding tools.
BMad Method Open source

Full agile AI-driven development framework. 12+ specialized agents (PM, Architect, Developer, UX, Scrum Master), scale-adaptive planning, 34+ workflows. Heavier than spec-kit but more comprehensive.
Jarvis Methodology Ours

Our approach: spec-only repos, tiny PRs (1-2 lines), attention budget, safety by architecture. The spec IS the product.
